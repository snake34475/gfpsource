# 功夫派 (GFP) 游戏复刻技术文档

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 游戏名称 | 功夫派 (GongFuPai / Kung Fu School) |
| 开发商 | 淘米 (Taomee) |
| 游戏类型 | 2D 横版动作 MMORPG |
| 原运行平台 | Adobe AIR 桌面端 |
| 原编程语言 | ActionScript 3 (AS3) |
| 源码规模 | ~1,581 个 .as 文件 |

### 源码范围

| 模块 | 路径 | 说明 |
|------|------|------|
| **clientcoredll** | `clientcoredll/` | 核心库 SWF（加密/解密、Crossbridge C 模块桥接） |
| **clientappdll** | `clientappdll/` | 游戏应用主模块（~1,528 个 AS 文件，所有游戏内容） |
| **xml 配置** | `xml/` | 服务器配置、Bean 注册表 |

### 缺失部分

- 后端服务器源码（战斗伤害计算、属性系统、掉落逻辑等）
- `com.gfp.core.*` 核心游戏引擎框架库
- `com.taomee.*` 淘米基础框架库
- 服务器端数值配置（怪物属性表、技能数值表等）

## 二、架构分析

### 2.1 客户端-服务器架构

```
┌─────────────────────────────────────────────────────────┐
│                      服 务 端                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐ │
│  │ 伤害计算  │  │ 属性系统  │  │ 掉落逻辑  │  │ 验证逻辑 │ │
│  │ (攻-防)  │  │ (攻击/防御)│  │ (物品/经验)│  │ (反作弊) │ │
│  └──────────┘  └──────────┘  └──────────┘  └─────────┘ │
│                        ↕ Socket TCP (端口 1863)         │
├─────────────────────────────────────────────────────────┤
│                      客 户 端                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐ │
│  │ 输入处理  │  │ 动画播放  │  │ UI 显示   │  │ 音效播放 │ │
│  │ (按键/鼠标)│  │ (受击/移动)│  │ (血条/数字)│  │ (打击音效)│ │
│  └──────────┘  └──────────┘  └──────────┘  └─────────┘ │
└─────────────────────────────────────────────────────────┘
```

**核心结论**: 伤害计算公式（攻击力 - 防御力、暴击倍率等）**不在客户端**，而是由服务器计算后将结果下发给客户端显示。

### 2.2 网络通信协议

- **传输层**: Socket TCP 二进制协议
- **消息路由**: 通过 `CommandID` 常量标识消息类型
- **数据格式**: `ByteArray` 二进制序列化
- **Bean 注册**: `xml/dll.xml` 注册了 **60+ 个命令监听器**

### 2.3 核心模块

| 模块 | 目录 | 文件数 | 功能 |
|------|------|--------|------|
| 地图处理 | `mapProcess/` | 521 | 各地图专属逻辑 |
| 管理器 | `manager/` | 83 | 各系统管理器 |
| 命令监听 | `cmdl/` | 60 | 网络命令处理 |
| 战斗 | `fight/` | 30 | PVE/PVP 战斗 |
| 动画/过场 | `cartoon/` | 30 | 动画系统 |

```typescript
// types/battle.ts
interface BruiseInfo {
  decHP: number;        // 伤害值（服务器已计算好）
  hp: number;           // 受击后剩余HP
  atkerID: number;      // 攻击者ID
  userID: number;       // 受击者ID
  crit: boolean;        // 是否暴击
  critMultiple: number; // 暴击倍率
  decType: number;      // 伤害类型: 0=扣血, 1=吸收
  type: number;         // 受击动作类型: 0-8
  hitCount: number;    // 连击数
  breakFlag: boolean;  // 是否打断
  skillID: number;      // 技能ID
  skillLv: number;      // 技能等级
  runDuration: number; // 受击动作持续时间
  stiffDuration: number; // 僵直持续时间
  pos: { x: number, y: number }; // 受击位移目标位置
}

// decoder.ts
function decodeBruise(data: ByteArray): BruiseInfo {
  return {
    decHP: data.readInt(),
    hp: data.readInt(),
    atkerID: data.readUnsignedInt(),
    userID: data.readUnsignedInt(),
    crit: data.readBoolean(),
    critMultiple: data.readUnsignedByte(),
    decType: data.readUnsignedByte(),
    type: data.readUnsignedByte(),
    hitCount: data.readUnsignedByte(),
    breakFlag: data.readBoolean(),
    skillID: data.readUnsignedInt(),
    skillLv: data.readUnsignedByte(),
    runDuration: data.readFloat(),
    stiffDuration: data.readFloat(),
    pos: { x: data.readFloat(), y: data.readFloat() }
  };
}
```

### 4.3 Socket 通信模块

```typescript
// network/SocketClient.ts
export class SocketClient {
  private ws: WebSocket;
  private handlers: Map<number, (data: ByteArray) => void> = new Map();
  
  connect(host: string, port: number): Promise<void> {
    return new Promise((resolve, reject) => {
      this.ws = new WebSocket(`ws://${host}:${port}`);
      this.ws.onopen = () => resolve();
      this.ws.onerror = (e) => reject(e);
      this.ws.onmessage = (e) => this.handleMessage(e.data);
    });
  }
  
  send(cmdId: number, ...args: any[]): void {
    const packet = PacketEncoder.encode(cmdId, args);
    this.ws.send(packet);
  }
  
  on(cmdId: number, handler: (data: ByteArray) => void): void {
    this.handlers.set(cmdId, handler);
  }
  
  private handleMessage(buffer: ArrayBuffer): void {
    const { cmdId, data } = PacketDecoder.decode(buffer);
    const handler = this.handlers.get(cmdId);
    if (handler) handler(data);
  }
}
```

## 五、框架选型

### 5.1 推荐方案：TypeScript + Phaser 3

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 语言 | TypeScript | 与AS3语法高度相似，转换成本最低 |
| 渲染引擎 | Phaser 3 | 成熟2D引擎，支持Sprite/Animation/Tilemap |
| UI框架 | React + Canvas | 界面与游戏分离，便于维护 |
| 服务端 | Node.js/Go | 快速开发，便于处理TCP连接 |
| 数据库 | MySQL + Redis | 持久化 + 缓存 |

### 5.2 备选方案：Cocos Creator

- 适合偏UI的游戏
- 有可视化编辑器
- 支持热更新
- TypeScript原生支持

### 5.3 不推荐：Unity

- 2D支持较弱
- 资源转换成本高

## 七、关键技术难点

### 7.1 服务端逻辑重建

由于服务端源码缺失，需要重建：
- 伤害计算公式
- 属性系统（攻击/防御/生命等）
- 物品掉落概率
- 任务判定逻辑
- 经验获取公式

### 7.2 动画系统

原版使用Flash动画，需要：
- 提取为序列帧
- 或转换为Lottie格式
- 或使用DragonBones

### 7.3 加密协议

`clientcoredll` 中包含加密模块，可能需要：
- 分析MEncrypt算法
- 或绕过客户端加密（模拟服务器）
